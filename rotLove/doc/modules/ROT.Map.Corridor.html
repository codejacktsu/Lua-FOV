Corridor:isValid (gen, isWallCallback, canBeDugCallback)
Use two callbacks to confirm corridor validity.

Parameters:

  • gen userdata The map generator calling this function. Lack of bind() function requires this. This is mainly so the map generator can hava a self reference in the two callbacks.
  • isWallCallback function A function with three parameters (gen, x, y) that will return true if x, y represents a wall space in a map.
  • canBeDugCallback function A function with three parameters (gen, x, y) that will return true if x, y represents a map cell that can be made into floorspace.

Returns:

    boolean true if corridor is valid.
Corridor:create (gen, digCallback)
Create. Function runs a callback to dig the corridor into a map

Parameters:

  • gen userdata The map generator calling this function. Passed as self to the digCallback
  • digCallback function The function responsible for digging the corridor into a map.
Corridor:createPriorityWalls (gen, priorityWallCallback)
Mark walls as priority for a future feature. Use this for storing the three points at the end of the corridor that you probably want to make sure gets a room attached.

Parameters:

  • gen userdata The map generator calling this function. Passed as self to the digCallback
  • priorityWallCallback function The function responsible for receiving and processing the priority walls
